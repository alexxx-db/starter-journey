# Before you Start

> **You'll learn** the foundational Databricks concepts and pick your cloud deployment model in ~20 min.
>
> **Prereqs:** none

## Why this matters

Deploying Databricks infrastructure without understanding the account model, workspace isolation, and governance layer leads to rework. Teams that skip this section typically hit permission issues, data-access surprises, or tenant-layout regrets within the first month.

Spend 20 minutes here. It pays back every time you avoid a misconfigured workspace or an ungoverned data path.

## What you'll cover

### Foundations

Core concepts you need before touching any cloud console:

- **[Account Console](/docs/before-you-start/foundations/account-console)** — The central admin portal for workspaces, users, and billing.
- **[Workspace](/docs/before-you-start/foundations/workspace)** — The environment where teams build and run data + AI workloads.
- **[Unity Catalog](/docs/before-you-start/foundations/unity-catalog)** — Centralized governance for all data and AI assets.
- **[Recap and learning](/docs/before-you-start/foundations/recap-and-learning)** — Quick-check questions to confirm you're ready.

### Cloud tenant ready

Decide how Databricks maps to your cloud accounts:

- **[Single-tenant setup](/docs/before-you-start/cloud-tenant-ready/single-tenant-setup)** — One cloud account, all Databricks resources in one place.
- **[Multi-tenant setup](/docs/before-you-start/cloud-tenant-ready/multi-tenant-setup)** — Multiple cloud accounts separated by environment or business unit.
